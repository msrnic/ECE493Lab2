## Use Case 7: Receive Review Invitation

**Goal in Context**: Notify reviewers of assignments  
**Scope**: Conference Management System  
**Level**: User Goal  
**Primary Actor**: Reviewer  
**Secondary Actors**: Notification Service  
**Trigger**: Reviewer is assigned to a paper  

## Success End Condition
* Reviewer is notified

## Failed End Condition
* Reviewer is not informed

## Preconditions
* Reviewer is registered

## Main Success Scenario
1. System sends notification
2. Reviewer receives invitation

## Extensions
* **1a**: Notification delivery failure  
  * **1a1**: System records failure evidence and schedules retry every 5 minutes
  * **1a2**: System retries up to 3 times; on success, system marks invitation delivered and stops retries
  * **1a3**: If all retries fail, system marks invitation failed and flags manual follow-up
* **1b**: Reviewer assignment is removed while invitation is pending/retrying
  * **1b1**: System cancels the invitation and stops all pending retries immediately
* **2a**: Delivery failure evidence is requested
  * **2a1**: System allows access only to paper editors and support/admin roles
  * **2a2**: System denies access to other authenticated users
* **1c**: Duplicate assignment event occurs for same reviewer-paper assignment
  * **1c1**: System reuses existing active invitation and does not create a duplicate active invitation

## Related Information
* **Priority**: High  
* **Frequency**: Moderate  
* **Open Issues**: Notification preferences
