## Use Case 13: Generate Conference Schedule

**Goal in Context**: Automatically organize sessions  
**Scope**: Conference Management System  
**Level**: User Goal  
**Primary Actor**: Administrator  
**Trigger**: Administrator initiates scheduling  

## Success End Condition
* Schedule is generated

## Failed End Condition
* Schedule generation fails

## Preconditions
* Accepted papers exist

## Main Success Scenario
1. Administrator initiates process
2. System assigns sessions
3. System generates schedule

## Extensions
* **1a**: Non-administrator initiates scheduling  
  * **1a1**: System rejects request as unauthorized
* **2a**: Conflict detected  
  * **2a1**: System flags issue and continues generation
* **2b**: Generation already in progress  
  * **2b1**: System rejects new request with in-progress message
* **2c**: Conflict review requested
  * **2c1**: Administrator or Editor can view full conflict details for follow-up
* **3a**: Required inputs missing  
  * **3a1**: System fails generation with clear reason
* **3b**: Generation succeeds  
  * **3b1**: System stores new schedule version and marks it latest active
