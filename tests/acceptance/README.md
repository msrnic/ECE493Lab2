# Acceptance Evidence Conventions

For UC-12 evidence files:

- `UC-12-happy-path.log`: Trigger + first-attempt delivery evidence.
- `UC-12-retry.log`: Initial failure + retry evidence.
- `UC-12-failure-log.log`: Unresolved failure + admin visibility evidence.
- `UC-12-performance.log`: Latency and delivery-rate evidence.

Naming convention: `UC-12-<scenario>.log`.
