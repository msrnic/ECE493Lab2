## Use Case 8: Access Assigned Paper

**Goal in Context**: Allow reviewers to read assigned submissions  
**Scope**: Conference Management System  
**Level**: User Goal  
**Primary Actor**: Reviewer  
**Trigger**: Reviewer selects assigned paper  

## Success End Condition
* Paper content is accessible

## Failed End Condition
* Paper cannot be accessed

## Preconditions
* Reviewer accepted assignment

## Main Success Scenario
1. Reviewer logs in
2. Reviewer selects paper
3. System displays paper files

## Extensions
* **3a**: Access revoked  
  * **3a1**: System denies access
* **3b**: Session expired before file retrieval  
  * **3b1**: System requires re-authentication and does not display protected files
* **3c**: Files temporarily unavailable  
  * **3c1**: System shows temporary unavailability and allows one immediate retry  
  * **3c2**: Additional repeated retries are throttled to one request every 5 seconds per reviewer-paper
* **3d**: Revocation occurs during active viewing  
  * **3d1**: Already displayed content remains visible  
  * **3d2**: Every new file request is denied immediately
* **3e**: Access outcomes are reviewed  
  * **3e1**: Paper editors and support/admin can view access-attempt records  
  * **3e2**: Non-authorized users are denied access to records
  * **3e3**: Expired sessions are denied access to records until re-authentication

## Related Information
