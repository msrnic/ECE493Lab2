# Conference Management System — Use Case

## Use Case 5: Save Paper Draft

**Goal in Context**: Allow authorized users to save, resume, and restore incomplete submissions without data loss  
**Scope**: Conference Management System  
**Level**: User Goal  
**Primary Actor**: Author  
**Supporting Actors**: Conference Administrator  
**Trigger**: Author selects “Save Draft”  

## Success End Condition
* Latest draft state is saved and available for later editing with version history intact

## Failed End Condition
* Save or restore request is rejected and previously saved draft state remains unchanged

## Preconditions
* Author has started a submission
* Submission status is in progress
* User is authenticated

## Main Success Scenario
1. Author enters or updates submission metadata and optional files.
2. Author selects save option.
3. System validates authorization and base revision.
4. System stores an immutable draft version and updates latest revision.
5. System confirms the save outcome.

## Extensions
* **3a**: System error  
  * **3a1**: Draft is not saved and an explicit retry message is shown
* **3b**: Stale revision is detected  
  * **3b1**: Save is rejected and user must reload latest draft before retrying
* **5a**: User returns later to continue work  
  * **5a1**: System loads latest saved draft by default
* **5b**: Owner or conference administrator chooses an earlier version  
  * **5b1**: System restores the selected version as a new latest immutable version
* **5c**: Non-owner and non-administrator attempts to view or restore versions  
  * **5c1**: System denies access
* **5d**: Final submission is completed  
  * **5d1**: System retains only the latest draft version and prunes older versions
